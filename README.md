# Basic MMO RPG

Стартовый 2D online RPG / UO-like prototype на Python.

Текущий этап: MVP 10, где несколько клиентов подключаются к authoritative websocket-серверу, входят по имени персонажа, видят друг друга, общаются в чате, восстанавливают сохраненную позицию после reconnect-а, взаимодействуют с NPC и lootable-объектами, хранят инвентарь, экипируют предмет в слот `main_hand`, проходят первые gathering-петли, взаимодействуют с живой овцой Барбарой в загоне, атакуют тренировочный манекен и сражаются с первым non-aggro противником.

## Быстрый запуск

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -e ".[dev]"
```

В первом терминале запусти сервер:

```powershell
python -m basic_mmo_rpg.server.app
```

Во втором и третьем терминалах запусти клиентов:

```powershell
python -m basic_mmo_rpg.client.app --server ws://127.0.0.1:8765 --name Alice
python -m basic_mmo_rpg.client.app --server ws://127.0.0.1:8765 --name Bob
```

Управление: `WASD` или стрелки. `Enter` открывает ввод чата, повторный `Enter` отправляет сообщение, `Esc` отменяет активный ввод. `J` показывает/скрывает журнал последних сообщений текущей сессии. `B` показывает/скрывает инвентарь и paperdoll. В панели `B` клик по экипируемому предмету берет его в руку, клик по занятому слоту `В руке` снимает предмет. `Tab` включает/выключает боевой режим; при выключении серверный auto-attack сбрасывается. Наведение курсора на NPC, овцу или lootable-объект показывает имя объекта, наведение на воду, дерево или камень подсвечивает ресурсный тайл, `F` отправляет запрос взаимодействия с объектом или ресурсным тайлом строго под курсором. В боевом режиме attackable-объект под курсором подсвечивается красным, клик по нему выбирает цель auto-attack, а выбранная цель отмечается темно-красным перекрестием. Клик по другому персонажу показывает его никнейм на 3 секунды. При смерти открывается окно `Вы погибли` с кнопкой `Возродиться`. Закрытие клиента сейчас только через крестик окна. Остановка сервера: `Ctrl+C`.

На стартовой карте внутри помещения стоят NPC `Funday`, `Jack Lumber` и `Kopai`. Если персонаж находится не дальше 64 пикселей и курсор наведен на NPC, реплика появляется над NPC и попадает в журнал сообщений. Если нужного tutorial-инструмента еще нет, сервер добавляет его в persistent-инвентарь и отправляет клиенту `inventory_updated`. NPC не экипируют инструмент автоматически: игрок сам берет его в руку через панель `B`.

Рыбалка работает через водные тайлы: если курсор наведен на воду рядом с персонажем, клавиша `F` отправляет `interact_requested` с координатами тайла. Без удочки в руке сообщение `Нужна удочка в руке` появляется в нижнем логе событий без записи в журнал. С экипированной удочкой персонаж на 2 секунды входит в состояние подготовки: в это время он не двигается, не атакует, не взаимодействует и не меняет предмет в руке. После подготовки попытка имеет шанс успеха `0.5`; при успехе персонаж получает `Рыба`, при неудаче видит `Рыба сорвалась` в нижнем логе событий. Эти результаты попадают в журнал. Если принести `Funday` две рыбы, сервер атомарно списывает `Рыба x2` и выдает `Gold x1`.

Рубка работает через тайлы деревьев: без `Топор для рубки` в руке сообщение `Нужен топор в руке` появляется в нижнем логе событий без записи в журнал. С экипированным топором после общей 2-секундной подготовки попытка всегда успешна, дает `Бревно`, пишет `Вы нарубили древесины` в журнал и показывает результат в нижнем логе событий. Если принести `Jack Lumber` пять бревен, сервер атомарно списывает `Бревно x5` и выдает `Gold x1`.

Добыча камня работает через тайлы `rock`: без `Кирка` в руке сообщение `Нужна кирка в руке` появляется в нижнем логе событий без записи в журнал. С экипированной киркой после общей 2-секундной подготовки попытка имеет шанс успеха `0.8`; при успехе персонаж получает `Камень`, при неудаче видит `Не удалось добыть камень` в нижнем логе событий. Если принести `Kopai` три камня, сервер атомарно списывает `Камень x3` и выдает `Gold x1`.

В правой верхней части карты есть загон с калиткой и овцой `Овца`; Fogu называет ее Барбарой в своей реплике. Калитка переключается клавишей `F` под курсором: закрытая калитка является solid-сущностью, открытая становится проходимой для игрока и овцы. Овца имеет `15 HP`, хранит состояние шерсти и каждые 2 секунды выбирает направление: если соседний тайл свободен, она плавно идет туда 2 секунды, иначе стоит на месте 2 секунды; затем ждет еще 2 секунды до следующей попытки. NPC `Fogu` выдает `Ножницы`, если их еще нет. С ножницами в руке можно состричь с овцы `Шерсть`; через 30 секунд сервер снова помечает шерсть как выросшую. `Fogu` меняет `Шерсть x1` на `Gold x1`.

В нижней правой части карты стоит `Тренировочный манекен`. Это component-based world-object с независимыми `lootable`, `combat` и `respawn` компонентами: при первом взаимодействии через `F` сервер persistent-записывает loot-claim персонажа, добавляет в инвентарь `Ржавый меч` и отправляет результат `Вы вытащили Ржавый меч из манекена`, который попадает в журнал. Повторные взаимодействия этого же персонажа с манекеном ничего не делают. Loot доступен независимо от того, целый манекен или разрушенный. `Ржавый меч` можно взять в руку через панель `B`.

Базовая PvE-боевка работает через `Tab` и клик по attackable-объекту. Сервер запоминает цель и выполняет auto-attack authoritative: если персонаж вне дистанции 64 px, атака молчит и ждет возвращения в радиус; если цель в радиусе, удар проходит по cooldown. Без подходящего оружия используется unarmed-урон `1-2`, `Ржавый меч` бьет на `3-6` с cooldown `1.2s`, шанс попадания пока фиксированный `85%`. Каждый hit/miss отправляется как `combat_event` и попадает в журнал: `Вы атаковали Тренировочный манекен: -4` или `Вы промахнулись`. При `0 HP` манекен становится `Разрушенный тренировочный манекен`, остается solid, auto-attack сбрасывается, а через 10 секунд server-time восстанавливается.

В левом нижнем углу карты гуляет `Кабан`. Пока его не атаковали, он ведет себя как neutral creature; после любой атаки вступает в бой, преследует атакующего, но срывает погоню при слишком большом удалении от дома. Кабан атакует на дистанции 64 px, имеет собственные hit chance, урон и cooldown, а цель выбирает по наибольшему нанесенному ему урону. При смерти кабан скрывается, через 60 секунд respawn-ится дома с 50% HP, а на земле остается проходимый и не attackable труп на 5 минут. Первый персонаж, взаимодействующий с трупом, получает `Кожа x2`. У игрока есть `30 HP`; при смерти он становится невидимым, не может двигаться, атаковать или взаимодействовать, видит `Вы погибли` и возрождается кнопкой у крестa на карте с `15 HP`.

По умолчанию сервер хранит позиции персонажей в `data/game.sqlite3`. Локальные SQLite-файлы игнорируются git-ом.

```text

Working directory: <project root>
```

```text
Script path: <project root>\scripts\run_server.py
Working directory: <project root>
Python interpreter: <project root>\.venv\Scripts\python.exe
```

## Редактор карты

Редактор карты живет отдельно от игрового клиента и сервера:

```powershell
.\.venv\Scripts\python.exe scripts\run_map_editor.py --map assets\maps\starter_map.json
```

Управление:

- `F1` - показать/скрыть окно hotkey'ев;
- `1..9` - выбрать тайл из палитры;
- ЛКМ по карте - рисовать выбранным тайлом;
- удержание ЛКМ - рисовать кистью 1x1;
- ПКМ по карте - выбрать тайл под курсором;
- ЛКМ по entity - выбрать и перетащить entity;
- `Ctrl` при перетаскивании entity - притянуть центр entity к центру тайла;
- колесо мыши - вертикальная прокрутка карты;
- `Shift` + колесо мыши - горизонтальная прокрутка карты;
- стрелки или `WASD` - прокрутка карты;
- средняя кнопка мыши - перетащить видимую область карты;
- `Z` - переключить масштаб `1x` / `0.5x`;
- `Ctrl+S` - сохранить карту.

Перед первой записью за сессию редактор создает backup рядом с картой:
`starter_map.json.bak`, `starter_map.json.bak.1` и так далее. Backup-файлы игнорируются git-ом.
Сохранение проходит через загрузчик карты, поэтому карта с entity на solid-тайле или за границей мира не будет записана.

Новая карта произвольного размера создается командой:

```powershell
.\.venv\Scripts\python.exe scripts\run_map_editor.py --map assets\maps\my_map.json --new 60x35
```

Новая карта берет `legend` и entities из `starter_map.json`, заполняет все поле травой `.` и открывается в редакторе.
Внешняя стена не создается: границей мира остается размер карты. Все entities, кроме gate, ставятся в левом верхнем углу,
чтобы их можно было потом перетащить на нужные места. Если файл уже существует, редактор не перезапишет его без флага
`--overwrite`.

## Структура

```text
src/basic_mmo_rpg/
  client/   pygame-ce: ввод, камера, рендер, UI, websocket-клиент
  domain/   чистая игровая логика: карта, геометрия, сущности мира, инвентарь, движение, коллизии
  server/   asyncio/websockets authoritative-сервер и серверное состояние мира
  shared/   JSON-протокол сообщений между клиентом и сервером
  storage/  загрузка JSON-карты и сохранение персонажей в SQLite
assets/maps/
  starter_map.json
scripts/
  run_client.py
  run_server.py
tests/
```

## Сетевая модель

Клиент сначала отправляет имя персонажа, а дальше отправляет намерения движения, сообщения чата и запросы взаимодействия:

```text
client -> server: join_requested
client -> server: move_requested
client -> server: chat_sent
client -> server: interact_requested
client -> server: equip_item_requested
client -> server: unequip_item_requested
client -> server: attack_requested
client -> server: stop_attack_requested
client -> server: respawn_requested
```

Сервер хранит активные сессии, кикает старое подключение при повторном входе тем же именем, применяет движение через доменную функцию `move_player`, проверяет коллизии с тайлами и runtime-объектами, двигает овцу и neutral/enemy creature-сущности, проверяет дистанцию взаимодействия с NPC, калиткой, creature-сущностями, lootable-объектами и ресурсными тайлами, выполняет правила инвентаря, обмена, одноразового loot-а и PvE-боевки, сохраняет позиции, инвентарь, экипировку и persistent loot-claims в SQLite и рассылает snapshot мира:

```text
server -> client: connection_accepted
server -> client: world_snapshot
server -> client: chat_message
server -> client: interaction_result
server -> client: inventory_updated
server -> client: equipment_updated
server -> client: combat_event
server -> client: entity_removed
```

Клиент не решает, где реально находится персонаж в online-режиме, что экипировано и была ли успешной добыча; он рисует состояние, полученное от сервера.
