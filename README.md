# Basic MMO RPG

Стартовый 2D online RPG / UO-like prototype на Python.

Текущий этап: MVP 7, где несколько клиентов подключаются к authoritative websocket-серверу, входят по имени персонажа, видят друг друга, общаются в чате, восстанавливают сохраненную позицию после reconnect-а, взаимодействуют с NPC, хранят инвентарь, экипируют инструмент в слот `main_hand` и проходят первые маленькие gathering-петли с рыбалкой, рубкой деревьев и добычей камня.

## Быстрый запуск

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -e ".[dev]"
```

В первом терминале запусти сервер:

```powershell
python -m basic_mmo_rpg.server.app
```

Во втором и третьем терминалах запусти клиентов:

```powershell
python -m basic_mmo_rpg.client.app --server ws://127.0.0.1:8765 --name Alice
python -m basic_mmo_rpg.client.app --server ws://127.0.0.1:8765 --name Bob
```

Управление: `WASD` или стрелки. `Enter` открывает ввод чата, повторный `Enter` отправляет сообщение, `Esc` отменяет активный ввод. `J` показывает/скрывает журнал последних сообщений текущей сессии. `B` показывает/скрывает инвентарь и paperdoll. В панели `B` клик по инструменту экипирует его в руку, клик по занятому слоту `В руке` снимает предмет. Наведение курсора на NPC показывает его имя, наведение на воду, дерево или камень подсвечивает ресурсный тайл, `F` отправляет запрос взаимодействия с объектом или ресурсным тайлом строго под курсором. Клик по другому персонажу показывает его никнейм на 3 секунды. Закрытие клиента сейчас только через крестик окна. Остановка сервера: `Ctrl+C`.

На стартовой карте внутри помещения стоят NPC `Funday`, `Jack Lumber` и `Kopai`. Если персонаж находится не дальше 64 пикселей и курсор наведен на NPC, реплика появляется над NPC и попадает в журнал сообщений. Если нужного tutorial-инструмента еще нет, сервер добавляет его в persistent-инвентарь и отправляет клиенту `inventory_updated`. NPC не экипируют инструмент автоматически: игрок сам берет его в руку через панель `B`.

Рыбалка работает через водные тайлы: если курсор наведен на воду рядом с персонажем, клавиша `F` отправляет `interact_requested` с координатами тайла. Без удочки в руке над персонажем появляется пузырь `Нужна удочка в руке` без записи в журнал. С экипированной удочкой попытка имеет шанс успеха `0.5` и cooldown 1 секунду; при успехе персонаж получает `Рыба`, при неудаче видит `Рыба сорвалась`. Эти результаты попадают в журнал. Если принести `Funday` две рыбы, сервер атомарно списывает `Рыба x2` и выдает `Gold x1`.

Рубка работает через тайлы деревьев: без `Топор для рубки` в руке над персонажем появляется `Нужен топор в руке` без записи в журнал. С экипированным топором попытка всегда успешна, дает `Бревно` и пишет `Вы нарубили древесины` в журнал. Если принести `Jack Lumber` пять бревен, сервер атомарно списывает `Бревно x5` и выдает `Gold x1`. Gathering cooldown сейчас общий для всех видов добычи.

Добыча камня работает через тайлы `rock`: без `Кирка` в руке над персонажем появляется `Нужна кирка в руке` без записи в журнал. С экипированной киркой попытка имеет шанс успеха `0.8`; при успехе персонаж получает `Камень`, при неудаче видит `Не удалось добыть камень`. Если принести `Kopai` три камня, сервер атомарно списывает `Камень x3` и выдает `Gold x1`. Gathering cooldown общий для рыбалки, рубки и добычи камня.

По умолчанию сервер хранит позиции персонажей в `data/game.sqlite3`. Локальные SQLite-файлы игнорируются git-ом.

## Запуск из PyCharm

Для клиента можно создать Python-конфигурацию:

```text
Script path: <project root>\scripts\run_client.py
Working directory: <project root>
Python interpreter: <project root>\.venv\Scripts\python.exe
Parameters: --server ws://127.0.0.1:8765 --name Alice
```

Для сервера:

```text
Script path: <project root>\scripts\run_server.py
Working directory: <project root>
Python interpreter: <project root>\.venv\Scripts\python.exe
```

## Структура

```text
src/basic_mmo_rpg/
  client/   pygame-ce: ввод, камера, рендер, UI, websocket-клиент
  domain/   чистая игровая логика: карта, геометрия, сущности мира, инвентарь, движение, коллизии
  server/   asyncio/websockets authoritative-сервер и серверное состояние мира
  shared/   JSON-протокол сообщений между клиентом и сервером
  storage/  загрузка JSON-карты и сохранение персонажей в SQLite
assets/maps/
  starter_map.json
scripts/
  run_client.py
  run_server.py
tests/
```

## Сетевая модель

Клиент сначала отправляет имя персонажа, а дальше отправляет намерения движения, сообщения чата и запросы взаимодействия:

```text
client -> server: join_requested
client -> server: move_requested
client -> server: chat_sent
client -> server: interact_requested
client -> server: equip_item_requested
client -> server: unequip_item_requested
```

Сервер хранит активные сессии, кикает старое подключение при повторном входе тем же именем, применяет движение через доменную функцию `move_player`, проверяет коллизии с тайлами и объектами, проверяет дистанцию взаимодействия с NPC и ресурсными тайлами, выполняет правила инвентаря и обмена, сохраняет позиции и инвентарь в SQLite и рассылает snapshot мира:

```text
server -> client: connection_accepted
server -> client: world_snapshot
server -> client: chat_message
server -> client: interaction_result
server -> client: inventory_updated
server -> client: equipment_updated
server -> client: entity_removed
```

Клиент не решает, где реально находится персонаж в online-режиме, что экипировано и была ли успешной добыча; он рисует состояние, полученное от сервера.
