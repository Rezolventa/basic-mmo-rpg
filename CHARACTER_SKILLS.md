# Игровые скиллы персонажа

Скиллы являются server-authoritative persistent-состоянием персонажа и хранятся в SQLite. Клиент только отображает значения, полученные от сервера через `skills_updated`.

Значение скилла хранится в десятых долях процента: `0` означает `0.0%`, `400` означает `40.0%`, `1000` означает `100.0%`. Для demo-этапа действует общий cap прокачки `40.0%`.

При первом входе персонажа сервер лениво создает недостающие скиллы. Стартовое значение каждого скилла сейчас рандомизируется отдельно в диапазоне `0.0%`-`9.9%`.

## Прокачка

Скилл может вырасти только после валидного завершенного применения связанного действия. Для gathering-а это означает, что персонаж был жив, находился достаточно близко к тайлу, имел нужный инструмент в руке, действие завершилось и был сделан roll результата добычи.

Неуспешная добыча тоже считается валидным применением и может прокачать скилл. Отказы до roll-а результата не качают скилл: нет инструмента, персонаж умер, цель далеко, тайл изменился или действие не было начато.

За одно применение скилл может вырасти максимум на `0.1%`.

Шанс роста:

- при `15.0%` и ниже: `1.0`;
- после `15.0%` шанс линейно уменьшается;
- при `39.9%`: `0.25`;
- при `40.0%`: `0.0`.

Порядок расчета: сначала сервер считает результат действия по текущему значению скилла, затем отдельно делает roll на рост скилла.

Если скилл вырос, сервер отправляет `skills_updated` и feed/journal-сообщение вида `Рыбалка повысилась до 12.4%`.

## Mining

Применение: добыча камня и руды из тайлов `rock`.

Шанс успеха:

- при `0.0%` шанс неуспешной добычи `0.8`;
- к `40.0%` шанс неуспешной добычи линейно уменьшается до `0.3`.

Дроп при успехе:

- до `20.0%` можно добыть только `Камень`;
- начиная с `20.0%` появляется `Железная руда`;
- при `20.0%` шанс руды среди успешных добыч `0.1`;
- к `40.0%` шанс руды среди успешных добыч линейно растет до `0.5`.

Формула:

1. сервер делает roll успеха/неуспеха;
2. если добыча успешна, сервер делает roll типа ресурса.

## Lumberjacking

Применение: рубка дерева.

Шанс успеха:

- при `0.0%` шанс неуспешной добычи `0.7`;
- к `40.0%` шанс неуспешной добычи линейно уменьшается до `0.2`.

Количество ресурсов:

- до `15.0%` успешная рубка дает `Бревно x1`;
- начиная с `15.0%` появляется шанс получить `Бревно x2`;
- при `15.0%` шанс двойного дропа `0.1`;
- к `40.0%` шанс двойного дропа линейно растет до `0.5`.

Формула:

1. сервер делает roll успеха/неуспеха;
2. если рубка успешна, сервер делает roll количества бревен.

## Fishing

Применение: ловля рыбы в водных тайлах.

Шанс успеха:

- при `0.0%` шанс неуспешной добычи `0.7`;
- к `40.0%` шанс неуспешной добычи линейно уменьшается до `0.2`.

Скорость:

- при `15.0%` и ниже длительность вылавливания остается полной: `2.0s`;
- после `15.0%` длительность линейно уменьшается;
- к `40.0%` длительность становится `1.5s`.

Формула:

1. при старте действия сервер определяет длительность gathering-подготовки;
2. после завершения подготовки сервер делает roll успеха/неуспеха;
3. затем сервер делает roll прокачки Fishing.
